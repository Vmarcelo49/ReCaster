chore(cmake): remove dead CASTER_TARGET_PROCESS and CASTER_NET_PORT options

Both were placeholders from the original skeleton that were never
wired into the actual code.

== CASTER_NET_PORT (7500) ==
The comment said "UDP/TCP port the hook.dll ENet placeholder listener
binds to" but:
  - 7500 appears in ZERO source files (.cpp/.hpp)
  - The real default port is 46318, hardcoded in config.hpp:21
    (kDefaultPort = 46318, matching zzcaster)
  - The -DCASTER_NET_PORT=7500 macro was passed to both caster.exe
    and hook.dll targets but never read by any code

This was a leftover from the initial "Hello World" skeleton before
the netplay engine was ported.

== CASTER_TARGET_PROCESS ("MBAA.exe") ==
The comment said "Override at configure time with
-DCASTER_TARGET_PROCESS=foo.exe" but:
  - The macro was passed to caster.exe but never read
  - The actual target is hardcoded in config.hpp:36
    (kTargetProcess = "MBAA.exe")
  - main.cpp:198 logs kTargetProcess, not the macro
  - So -DCASTER_TARGET_PROCESS=foo.exe would be silently ignored

== Cleanup ==
CMakeLists.txt:
  - Removed both set(CASTER_* CACHE ...) option declarations
  - Removed both from target_compile_definitions() for caster and hook
  - Removed both from the configure-time STATUS summary
  - Updated header comment to reflect actual project (was still
    describing the "Hello World skeleton")
  - Added note pointing to config.hpp as the source of truth for
    target process and default port

scripts/build.sh:
  - Removed -DCASTER_TARGET_PROCESS="$TARGET_PROCESS" from cmake
    configure (was being ignored anyway)
  - Removed the now-unused TARGET_PROCESS variable

== Validation ==
Full clean build (rm CMakeCache.txt + ./scripts/build.sh) passes.
caster.exe + hook.dll link successfully. Release zip generated.
No behavior change — the macros were dead code.
