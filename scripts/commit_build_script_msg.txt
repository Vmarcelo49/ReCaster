chore(build): rewrite build.sh — clean, strip, zip to release/

Rewrote scripts/build.sh to be minimal and automated. Removed
build-local.sh (was sandbox-specific, redundant).

== Changes ==

scripts/build.sh (rewritten):
  - Removed verbose info dump (source dir, build dir, toolchain, build
    type, target process, parallel jobs — all irrelevant noise)
  - Removed toolchain verification block (cmake will fail clearly if
    the toolchain is missing)
  - Removed per-file "before/after" strip logging
  - Removed the "Build artifacts:" ls + file output block
  - Removed the "Done. Copy caster.exe + hook.dll..." final message
  - Kept the 4 essential steps: configure, build, strip, zip
  - Added step 4: zip both binaries to release/caster-<timestamp>.zip
  - Final output: just the zip path (for scripting)

scripts/build-local.sh (deleted):
  - Was specific to a sandbox with /home/z/mingw-prefix/ toolchain
  - No longer needed; build.sh works on any system with mingw-w64

.gitignore (new):
  - release/        (zips are build artifacts, not source)
  - caster/config.ini, debug.log, mapping.ini (runtime files)

== New build.sh behavior ==

  $ ./scripts/build.sh              # configure + build + strip + zip
  $ ./scripts/build.sh rebuild      # skip configure, build only

Output:
  build/bin/caster.exe   (stripped, ~5MB)
  build/bin/hook.dll     (stripped, ~4MB)
  release/caster-YYYYMMDD-HHMMSS.zip   (both binaries, ~3.5MB)

The script prints only the zip path on success — clean for CI/scripting.

== Tested ==
