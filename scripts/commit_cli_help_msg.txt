feat(cli): comprehensive --help + add -help and /? aliases

== Problem ==
The user reported that `caster.exe -h` and `caster.exe --help` produced
no visible output, and `caster.exe -help` (single-dash) was not
recognized at all. The help text existed but was minimal and didn't
explain the relay model, game setup, or config file layout.

== Root cause ==
1. `-help` (single-dash) was not a recognized alias — only `-h` and
   `--help` were. Users typing `-help` got silent failure (the arg was
   treated as unknown, throwing a runtime_error that printed to stderr
   but was easy to miss in Wine's verbose output).
2. The help text was terse — just listed flags without explaining the
   relay server model (no port forwarding needed), game setup (where
   to place MBAA.exe), config file location, or exit codes.

== Fix 1: Add help aliases ==
Now recognizes all common help flags:
  -h          (short, Unix-style)
  --help      (long, GNU-style)
  -help       (single-dash long, Java/Windows-style)
  /?          (Windows CMD-style)

Updated the header comment in cli_args.hpp to document all aliases.

== Fix 2: Comprehensive help text ==
Rewrote helpText() in cli_args.cpp to include:

  USAGE:               all 4 help aliases shown
  MODES:               clearer descriptions (offline single/two player,
                       relay+direct host, etc.)
  OPTIONS:             defaults explained (port 46318, delay=auto,
                       rollback=4, name from config.ini)
  PEER FORMAT:         host:port vs #ABCD with examples
  RELAY SERVERS:       explicitly states NO port forwarding required
                       (outbound TCP+UDP 3939 only, consumer firewalls
                       allow by default). Default relay address shown.
  GAME SETUP:          4-step MBAA.exe search order (config game_dir,
                       same folder, game/ subdir, CWD)
  CONFIG FILE:         path + section names
  LOGGING:             log file location (Windows + Linux/Wine paths)
  EXAMPLES:            12 examples covering all modes + option combos
  EXIT CODES:          0/1/2 meanings
  VERSION:             caster 4.1-cpp, protocol v3, target game version

The RELAY SERVERS section is important — it reinforces the fix from
commit 9dcecf7 (remove incorrect port-forwarding advice) by explicitly
stating the relay handles NAT traversal automatically.

== Testing ==
All 4 aliases tested via Wine on the remote machine:
  wine caster.exe -h       → prints help, exits 0 ✓
  wine caster.exe --help   → prints help, exits 0 ✓
  wine caster.exe -help    → prints help, exits 0 ✓ (was broken before)
  wine caster.exe /?       → prints help, exits 0 ✓

Build passes clean (cmake --build, exit 0, no warnings).
